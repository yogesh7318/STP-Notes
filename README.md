## Why This Repository Exists

Most Git tutorials explain commands.

This lab teaches workflow.

It focuses on how real teams:

* Maintain clean commit history
* Work safely using branches
* Collaborate using Pull Requests
* Use CI pipelines as quality gates
* Recover safely from mistakes

The goal is not theory.
The goal is confidence in real-world development flow.

---

## What This Covers

The lab progresses from fundamentals to structured team workflow.

### Core Foundations

* Git installation & configuration
* Local repositories & commit discipline
* Understanding staging & diffs

### Collaboration Workflow

* Remote repositories with GitHub
* Branching strategy
* Merging (fast-forward & merge commit)
* Merge conflict resolution

### Team Simulation

* Push & Pull synchronization
* Pull Request lifecycle
* Code review concepts

### CI Automation

* GitHub Actions workflow setup
* Triggering pipelines on push
* Reading CI logs
* Understanding success vs failure signals

### Recovery & Safety

* Undo before commit
* Soft reset vs hard reset
* Using `git reflog` to recover deleted commits
* Understanding HEAD movement

## Industry Relevance

This lab mirrors real team practices:

* No direct push to main
* Structured Pull Requests
* CI as automated validation
* Clean, readable commit history
* Safe recovery during mistakes

These are the same fundamentals used in production environments.

After completing this lab, a learner can:

* Work confidently with Git from the command line
* Explain branching and merge behavior clearly
* Handle merge conflicts calmly
* Understand when and why CI runs
* Recover from common Git mistakes without panic
